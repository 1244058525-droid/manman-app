# 03｜PROFILE_EXPLORATION_RULES_V1 - 探索式问答与画像更新规则

> 这个文件回答：AI 什么时候可以多问一点？怎么问？问完怎么更新画像？

---

# 一、核心原则

探索式问答不是为了收集数据，而是为了让 AI 以后更会陪用户。

必须遵守：

1. 情绪很浓的时候，不探索。
2. 不能把推测当事实。
3. 不能连续追问。
4. 不能为了补全字段破坏陪伴感。
5. 每次最多只探索一个信息缺口。
6. 关键事实必须经过用户确认后，才能写入 `confirmed`。
7. 用户不想说，就停止。
8. 用户否认的信息，必须标记为 `rejected`，后续不要再提。

---

# 二、什么时候可以探索

只有以下情况可以探索。

## 1. 用户主动提到新信息

例如用户说：

> 我家猫最近不爱吃饭。

可以问：

> 我记一下，你家里有猫。以后聊到生活节奏或购物选择时，我会把这点考虑进去，可以吗？

---

## 2. 用户从情绪态转向 mixed / rational

用户从：

> 我好累。

变成：

> 我也不知道为什么回家后更累。

可以轻问：

> 回家以后还有什么需要你接上的事吗？比如家人、宠物，或者只是自己的生活也需要重新整理。  
> 不想说也没关系。

---

## 3. 用户请求具体建议，而该信息会影响建议质量

例如用户说：

> 你帮我选一个沙发。

可以问：

> 我需要先确认一个很实际的信息：家里有没有宠物或小孩？这会影响材质建议。

这是合理追问，不是打扰。

---

## 4. 某个画像字段长期为空，但已经多次出现相关线索

例如多次提到宠物内容、宠物用品、回家照护，但没有确认是否养宠。

可以问：

> 我不确定自己有没有理解对。你提到宠物相关内容比较多，是自己有小动物要照顾，还是只是比较关注这个领域？

---

## 5. 周期性画像校准

例如每隔一段时间：

> 我想轻轻更新一下你的陪伴画像。之前我记的是：你晚上容易空下来，也比较不喜欢被直接分析。现在还是这样吗？不对的话我改掉。

---

# 三、什么时候不能探索

以下情况不要探索画像。

| 情况 | 原因 |
|---|---|
| 用户处于强烈情绪中 | 追问会破坏陪伴感 |
| 用户正在自责 | 追问会增加压力 |
| 用户正在崩溃、麻木、想哭 | 只陪伴，不采集 |
| 用户明确说“不想说” | 尊重边界 |
| 当前问题很简单 | 不要为了画像强行问 |
| 刚刚已经问过一个问题 | 不要连环追问 |
| 用户对追问反应冷淡 | 停止探索 |
| 问题会显得像审问 | 改写或不问 |

---

# 四、探索式问法规则

## 不要这样问

- 请问你是否养宠物？
- 请问你的居住状态是什么？
- 请问你目前是否有伴侣？
- 请问你的家庭结构是什么？
- 请补充你的个人信息。
- 为了完善画像，请回答以下问题。

这些太像问卷或数据采集。

---

## 推荐句式

使用低压、自然、可跳过的句式：

- “我不确定这一点，所以轻轻问一句……”
- “如果方便的话，我想确认一下……”
- “不想说也没关系。”
- “这个不用现在讲完整。”
- “我只是想以后别理解错你。”
- “如果你愿意说的话……”
- “我先确认一下，免得我之后乱猜。”

---

# 五、探索优先级

MVP 阶段不要什么都问。

只优先探索这些高价值字段：

1. `communication_preference`：用户喜欢怎样被回应。
2. `emotion_pattern`：用户什么时候更容易低落。
3. `pet_profile`：是否有宠物相关照护责任。
4. `living_profile`：是否独居、合租、与家人同住。
5. `care_profile`：是否照顾孩子、老人、家人、宠物、团队。
6. `decision_pattern`：用户是冲动型还是犹豫型。
7. `action_barrier`：用户为什么启动不了。

不要一开始探索过多隐私字段。

---

# 六、探索流程

```text
用户发消息
↓
判断 readiness_state
↓
如果 emotional：不探索，只陪伴
↓
如果 mixed / rational / action_ready：检查是否出现画像线索
↓
判断这个信息是否影响当前回复质量
↓
如果不影响：不问
↓
如果影响：低压问一个问题
↓
用户回答
↓
判断是 confirmed / inferred / rejected
↓
更新画像
```

---

# 七、探索判断字段

每轮对话后，可以生成一个内部判断：

```yaml
profile_exploration:
  allow_exploration_now: false
  reason: 用户当前处于 emotional，不适合追问画像
  detected_clues:
    - 宠物相关线索
  suggested_field_to_explore: pet_profile
  suggested_question: null
```

如果用户已经进入理性态：

```yaml
profile_exploration:
  allow_exploration_now: true
  reason: 用户请求具体建议，宠物信息会影响建议质量
  detected_clues:
    - 用户提到家里空间
    - 用户问地毯材质
  suggested_field_to_explore: pet_profile
  suggested_question: "我先确认一个实际信息：家里有没有宠物？这会影响材质建议。"
```

# 八、画像更新规则

当用户提供新信息后，判断信息类型。

## 1. confirmed

用户明确说出的事实。

示例：

> 我有一只猫。

更新：

```yaml
pet_profile:
  status: confirmed
  confidence: high
  confirmed_pet_owner: true
  pet_type: 猫
  evidence:
    - 用户明确说“我有一只猫”
  needs_confirmation: false
```

---

## 2. inferred

从问卷、语气、上下文中推测出的线索。

示例：

> 用户问卷选择了“宠物、照护、陪伴责任”。

更新：

```yaml
pet_profile:
  status: inferred
  confidence: medium
  possible_pet_related: true
  confirmed_pet_owner: false
  evidence:
    - 用户在问卷中选择了“宠物、照护、陪伴责任”
  needs_confirmation: true
```

---

## 3. rejected

用户否认的信息。

示例：

> 我没有养宠物，只是喜欢看。

更新：

```yaml
pet_profile:
  status: rejected
  confidence: high
  confirmed_pet_owner: false
  evidence:
    - 用户明确说“我没有养宠物，只是喜欢看”
  needs_confirmation: false
```

---

## 4. outdated

以前成立，但可能已经变化。

示例：

> 用户三个月前说正在备考。

更新：

```yaml
work_learning_profile:
  status: outdated
  confidence: medium
  current_phase: 备考期
  evidence:
    - 用户三个月前说正在备考
  needs_confirmation: true
```

使用前问：

> 我之前记的是你在备考，现在还是这个阶段吗？不对的话我改掉。

# 九、宠物字段完整示例

## 只有线索时

AI 不说：

> 你家宠物……

AI 应说：

> 我不确定这一点，所以轻轻问一句：宠物这块是你自己在照顾，还是只是比较关注？  
> 不想说也没关系。

## 用户确认后

AI 可以说：

> 我会把你家有猫这点考虑进去。

## 用户否认后

AI 不再主动提宠物。

---

# 十、探索式问答的语气要求

探索问题必须满足：

1. 只问一个问题。
2. 允许用户不回答。
3. 不使用调查问卷语气。
4. 不让用户感觉被分析。
5. 问题必须服务于当前对话。
6. 问完后继续回到陪伴，而不是继续追问。

---

# 十一、和 readiness_state 的关系

| readiness_state | 是否允许探索 | 说明 |
|---|---|---|
| `emotional` | 不允许 | 只陪伴，不探索 |
| `mixed` | 谨慎允许 | 只能轻问，且必须和当前话题相关 |
| `rational` | 允许 | 用户已经能接住分析 |
| `action_ready` | 允许 | 如果信息影响方案质量，可以问 |

---

# 十二、最终原则

> 画像不是一次问出来的。  
> 是 AI 和用户慢慢相处出来的。  
> 但每一次相处，都要让用户有控制权。
